📘 Guia da Estrutura - Projeto TaskMaster

Este documento detalha a arquitetura do sistema desenvolvido, explicando a função de cada diretório e arquivo principal. O projeto foi construído seguindo o padrão MVC (Model - View - Controller), garantindo uma separação lógica e organizada do código.

1. 🧠 Back-end (A Lógica e o Cérebro)

Nesta camada residem as regras de negócio. O Controller processa as requisições enviadas pelo usuário através da interface.

Diretório: app/Http/Controllers/

  • AuthController.php: Gerencia a autenticação, incluindo login, registro de novos usuários e encerramento de sessão (logout).
  • DashboardController.php: Processa os dados consolidados (tarefas pendentes, concluídas, atrasadas) para exibição gráfica na tela inicial.
  • TrabalhoController.php: Controlador principal das tarefas (CRUD). Gerencia criação, edição, exclusão e listagem, além de aplicar filtros por categoria.
  • CommentController.php: Gerencia a persistência de comentários vinculados às tarefas.
  • ProfileController.php: Permite ao usuário logado gerenciar seus dados pessoais (nome, telefone e senha).

Diretório: app/Http/Requests/

  • TrabalhoRequest.php: Camada de validação. Assegura a integridade dos dados (datas válidas, campos obrigatórios) antes que cheguem ao Controller.

2. 🗄️ Banco de Dados (A Estrutura)

Camada responsável pela definição, estruturação e população dos dados.

Diretório: app/Models/ (Representação das Tabelas)

  • User.php: Representação do usuário no sistema.
  • Trabalho.php: Modelo principal das tarefas, contendo as relações com Categorias e Comentários.
  • Category.php: Define as categorias de tarefas e suas cores de exibição.
  • Comment.php: Representa a interação textual dentro de uma tarefa.

Diretório: database/migrations/

  • Funcionam como a "Planta Baixa" do banco de dados. Definem a estrutura das tabelas e colunas, permitindo o versionamento do banco e facilitando a implantação em diferentes ambientes através do comando migrate.

Diretório: database/seeders/ e factories/

  • Responsáveis pela automação e geração de dados falsos (fictícios) em português para testes de carga e visualização inicial do sistema.

3. 🎨 Front-end (O Visual)

Interface do usuário desenvolvida com o motor de templates Blade e estilizada com Bootstrap 5.

Diretório: resources/views/

  • layout.blade.php: O template mestre (esqueleto). Contém a Navbar, Rodapé e importações CSS/JS.
  • dashboard.blade.php: Painel de controle com estatísticas e gráficos de progresso (Relatório Consolidado).

Subdiretório: views/trabalho/

  • index.blade.php: Listagem principal de tarefas com filtros.
  • update.blade.php: Interface híbrida de edição de tarefa e chat de comentários.

4. 🛣️ Rotas (O Mapa)

Arquivo: routes/web.php

Define o mapeamento das URLs para os Controllers. Também gerencia os Middlewares de segurança, impedindo acesso não autorizado às áreas restritas do sistema.
